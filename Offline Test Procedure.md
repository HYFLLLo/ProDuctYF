## 1. 离线测试流程概述

### 1.1 流程目的

通过离线数据集测试，评估项目各模块是否达到预设的质量指标和性能要求，验证整体系统在脱离在线数据源环境下的工作能力。

### 1.2 流程范围

本文档定义离线测试的执行流程、测试规范、达标判定标准及测试周期，适用于：
- 事件理解分析Agent的离线评估
- 用户情绪/场景分析Agent的离线评估
- 决策层Agent的离线评估
- 业务价值指标的离线评估

### 2.2 Step 1: 数据准备

| 任务 | 详细说明 |
|------|---------|
| 数据导入 | 从test_data目录导入事件/用户/商家/Ground Truth数据集 |
| 格式校验 | 验证JSON格式、必填字段、数据类型 |
| 数据划分 | 按70:15:15比例划分训练集/验证集/测试集 |
| 数据预览 | 记录各类数据条数、字段完整性统计 |

### 2.3 Step 2: 事件理解分析Agent测试

| 任务 | 详细说明 |
|------|---------|
| 测试输入 | weather_test_data.json, sports_events_test_data.json, social_media_test_data.json |
| 执行处理 | 事件分类→结构化抽取→去重→热度计算 |
| 评估对比 | 对比输出与event_annotations.json中的标注数据 |
| 评估指标 | 分类准确率、信息抽取完整率、去重准确率、热度计算误差、处理延迟 |

### 2.4 Step 3: 用户情绪/场景分析Agent测试

| 任务 | 详细说明 |
|------|---------|
| 测试输入 | user_profiles.json, order_history_sample.json, scene_annotations.json |
| 执行处理 | 规则判断→场景验证推理→商品匹配推理 |
| 评估对比 | 对比输出与scene_annotations.json中的标注场景 |
| 评估指标 | 场景判断准确率、潜在需求命中率、召回率、处理延迟 |

### 2.5 Step 4: 决策层Agent测试

| 任务 | 详细说明 |
|------|---------|
| 测试输入 | 用户场景输出 + merchant_profiles.json + inventory_data.json |
| 执行处理 | 候选组合→销量预测→补货单→智能决策 |
| 评估对比 | 对比预测销量与ground_truth.json中的实际销量 |
| 评估指标 | 爆品预测准确率、补货建议准确率、缺货率降低、滞销率降低、决策延迟 |

### 2.6 Step 5: 结果汇总与达标判定

| 任务 | 详细说明 |
|------|---------|
| 结果汇总 | 汇总各Agent所有评估指标 |
| 达标判定 | 对照达标要求逐项判定 |
| 报告输出 | 生成标准化测试报告JSON |

## 5. 核心评估指标体系

### 5.1 事件理解分析Agent评估指标

| 指标 | 计算方式 | 达标要求 | 权重 |
|------|---------|---------|------|
| 分类准确率 | 正确分类事件数 / 已标注事件总数 | ≥90% | 25% |
| 分类错误率 | 错误分类事件数 / 已标注事件总数 | ≤5% | - |
| 信息抽取完整率 | 完整字段数 / 应有字段总数 | ≥95% | 20% |
| 去重准确率 | 正确去重数 / 应去重总数 | ≥98% | 15% |
| 热度计算误差 | \|预测热度-真实热度\| / 真实热度 | ≤15% | 25% |
| 处理延迟 | 全流程处理时间 | ≤5分钟 | 15% |

### 5.2 用户情绪/场景分析Agent评估指标

| 指标 | 计算方式 | 达标要求 | 权重 |
|------|---------|---------|------|
| 场景判断准确率 | 正确判断场景数 / 已标注场景总数 | ≥85% | 35% |
| 潜在需求命中率 | 命中潜在需求商品数 / 推荐商品总数 | ≥30% | 25% |
| 误判率 | 错误判断场景数 / 已标注场景总数 | ≤10% | - |
| 召回率 | 正确召回场景数 / 真实场景数 | ≥80% | 20% |
| 处理延迟 | 端到端处理时间 | ≤5秒/用户 | 20% |

### 5.3 决策层Agent评估指标

| 指标 | 计算方式 | 达标要求 | 权重 |
|------|---------|---------|------|
| 爆品预测准确率 | Top-N预测中实际爆品数 / N（N=10） | ≥75% | 30% |
| 补货建议准确率 | 补货建议与实际需求匹配度 | ≥80% | 25% |
| 缺货率降低 | (原始缺货率-预测后缺货率) / 原始缺货率 | ≥20% | 20% |
| 滞销率降低 | (原始滞销率-预测后滞销率) / 原始滞销率 | ≥15% | 15% |
| 决策生成延迟 | 单商家决策生成时间 | ≤3秒 | 10% |

### 5.4 业务价值评估指标

| 指标 | 计算方式 | 达标要求 |
|------|---------|---------|
| 缺货损失降低率 | (原缺货损失-现缺货损失) / 原缺货损失 | ≥25% |
| 滞销损耗降低率 | (原滞销损耗-现滞销损耗) / 原滞销损耗 | ≥20% |
| GMV提升率 | (预测后GMV-基准GMV) / 基准GMV | ≥10% |
| 客单价提升率 | (预测后客单价-基准客单价) / 基准客单价 | ≥8% |

## 6. 成功判定标准

### 6.1 单模块达标条件

| Agent | 关键指标（必须全部达标） | 加分指标（任意一项达标即加分） |
|-------|------------------------|------------------------------|
| 事件理解分析Agent | 分类准确率≥90% **且** 处理延迟≤5分钟 | 热度计算误差≤10% |
| 用户情绪/场景分析Agent | 场景判断准确率≥85% **且** 处理延迟≤5秒 | 潜在需求命中率≥40% |
| 决策层Agent | 爆品预测准确率≥75% **且** 补货准确率≥80% | 缺货损失降低率≥30% |

### 6.2 项目整体成功判定

**判定条件（必须全部满足）**：

1. **所有Agent关键指标100%达标**
2. **业务价值评估指标≥3/4达标**（缺货损失降低率、滞销损耗降低率、GMV提升率、客单价提升率中至少3项达标）
3. **端到端流程可正常运行**（无异常中断、日志完整）
4. **单商家完整决策流程≤30秒**（含三个Agent串行处理）

### 6.3 达标等级

| 等级 | 条件 |
|------|------|
| 优秀 | 所有指标100%达标 + 业务价值4/4达标 |
| 良好 | 所有关键指标达标 + 业务价值3/4达标 |
| 合格 | 所有Agent关键指标达标 |
| 不合格 | 任意一项Agent关键指标未达标 |
